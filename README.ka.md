# SupportTools

მოდულური CLI აპლიკაცია .NET-ის მრავალპროექტიანი გარემოს მართვისთვის:
Git სინქრონიზაცია, ბაზის სასიცოცხლო ციკლი, scaffolding, განთავსება და
მოშორებული სერვერის ოპერაციები.

[In English](README.md)

## აღწერა

SupportTools არის .NET 10 კონსოლის აპლიკაცია, რომელიც ახდენს
დაკავშირებული .NET პროექტების კოლექციის გარშემო განმეორებადი
ოპერაციული სამუშაოს ორკესტრაციას. იმის ნაცვლად, რომ თითო რეპოში
ჩახვიდე pull-ისთვის, თითო სერვისი ცალკე ააწყო, ან ხელით დაშიფრო
`appsettings.json` თითო environment-ისთვის, ყველაფერ ამას შენ აკეთებ
ერთი მენიუ-დაფუძნებული CLI-დან.

ხელსაწყო მუშაობს ნებისმიერ პლატფორმაზე, რომელიც მხარს უჭერს .NET-ს.
მოშორებული სერვერის სამუშაოს გადასცემს თანმდევი სერვისი სახელად
WebAgent, ამიტომ ერთი და იგივე SupportTools binary-ი დეველოპერის
მანქანაზე შეუძლია მართოს ინსტალაცია Linux-ზე, Windows-ზე, ან
ნებისმიერ ჰოსტზე, რომელზეც გაშვებულია WebAgent.

State — პროექტები, ჯგუფები, server ჩანაწერები, Git URL-ები — ცხოვრობს
ერთ JSON პარამეტრების ფაილში შენი user profile-ის ქვეშ.

## შესაძლებლობები

* **Git** — ბევრი რეპოზიტორიის ერთბაშად სინქი (ყველა პროექტი, ერთი
ჯგუფი, ან ერთი პროექტი); პროექტის Git URL-ების სიის მართვა
* **ბაზა** — dev ბაზის შექმნა ან წაშლა EF მიგრაციით; post-migration
სქემის კორექტირება; სქემის ანალიზი; scaffold prod-ის ასლიდან;
მონაცემების ექსტრაქცია JSON-ში; dev ბაზის seeding JSON-დან
* **განთავსება** — `appsettings.json`-ის შიფვრა, build-ის გამოქვეყნება,
პროგრამის ინსტალაცია ან წაშლა სერვერზე, სერვისების
გაშვება/გაჩერება/ვერსიის შემოწმება (ლოკალურად ან მოშორებულად
WebAgent-ით)
* **პროექტის შექმნა** — ახალი Console/API/Razor solution-ების bootstrap
შაბლონებიდან; პროექტის კონფიგურაციების import/export; NuGet
პაკეტების მასობრივი განახლება პროექტებში
* **კოდის ხარისხი** — `jb cleanupcode`-ის გაშვება solution-ზე; API-რაუტი
კონსტანტების კლასების გენერაცია კონტრაქტების მეტამონაცემებიდან
