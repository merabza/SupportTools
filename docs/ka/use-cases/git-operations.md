# Git ოპერაციები

SupportTools მართავს ბევრ პროექტს, რომელთაგან თითოეული შეიძლება
შედგებოდეს რამდენიმე Git რეპოზიტორიისგან. აქ მოცემული ბრძანებები
საშუალებას გაძლევენ შეუნახოთ ყველა ეს რეპო სინქრონიზებული, ხელით
თითო ფოლდერში გადასვლის გარეშე.

ყველა ბრძანება ვარაუდობს, რომ თითოეულ პროექტს აქვს Git რეპოზიტორიების
URL-ები კონფიგურირებული პროექტის `Git` ქვემენიუს მეშვეობით (იხ.
[კონფიგურაცია](../configuration.md)).

## მოკლე ცნობარი

|ბრძანება|არეალი|
|-|-|
|`Sync All Projects All Gits V2`|ყველა რეპოზიტორია ყველა პროექტში|

|`Git ScaffoldSeeder projects` ქვემენიუ|იგივე, მაგრამ პროექტის scaffold seeder რეპოებისთვის|

## როგორ მუშაობს სინქი

ყველა სინქის ბრძანება იყენებს ერთნაირ ხუთნაბიჯიან pipeline-ს
თითოეულ რეპოზე:

1. **Clone** — `git clone <url> <path>` თუ ფოლდერი არ არსებობს
2. **Stage და ინსპექცია** — `git add .`, შემდეგ `git status --porcelain`
3. **Commit** — თუ არის დაუკომიტებელი ცვლილებები, ეკითხება მესიჯს
(ხელახლა გამოიყენება batch-ში) და უშვებს `git commit`-ს
4. **Pull** — `git pull` თუ remote-ში ახალი commit-ებია
5. **Push** — `git push` თუ ლოკალურს აქვს commit-ები, რომლებიც remote-ში
არ არის

პროცესორი ანაწილებს თითოეულ რეპოს ოთხ მდგომარეობაში merge-base-ის
მეშვეობით: `UpToDate`, `NeedToPull`, `NeedToPush`, ან `Diverged`.

## თუ გჭირდება ყველა რეპოს ერთბაშად სინქი

გამოიყენე როცა დიდხანს არ ხართ კონტაქტში და გინდა ყველა პროექტი
განახლებული იყოს.

**ნაბიჯები**

1. `Main Menu` → `Sync All Projects All Gits V2`
2. შეიყვანე commit მესიჯი (გამოიყენება ლოკალური ცვლილებების მქონე
რეპოებისთვის)

**მოსალოდნელი შედეგი**

* თითოეული რეპო აცხადებს დასაწყის და საბოლოო მდგომარეობას
* ცვლილებების გარეშე რეპოები მთავრდება მყისიერად
* diverged ისტორიის რეპოები ცდება შეცდომით — საჭიროებენ ხელით
გადაწყვეტას

**ხშირი შეცდომები**

* **Diverged branch-ები.** თუ რეპოს აქვს ლოკალური და remote
commit-ები ერთდროულად, ხელსაწყო ჩერდება შეცდომით. არ არის
ავტომატური merge ან rebase. გადაწყვიტე ხელით დაზარალებულ რეპოში
ხელახალი გაშვების წინ.
* **არ არის ჩაკეტვა რეპოებს შორის.** თუ ორი სინქის გაშვება გადაიფარება
(მაგ. ორი shell), ერთი URL-ის რეპოები შეიძლება დააფრქვან. არ
გაუშვა სინქი ერთდროულად.
* **გამეორებული commit მესიჯი.** ერთი მესიჯი გამოიყენება batch-ში
ყველა რეპოზე. თუ გინდა თითო რეპოს ცალკე მესიჯი, ერთიდან-ერთი სინქი
გამოიყენე.
* **ჩუმი auth ჩავარდნა.** ხელსაწყო ეყრდნობა OS-ის Git credentials-ს
(SSH key, credential manager). თუ auth ჩავარდა, output არის ზოგადი
"cannot clone/push" — ჯერ შეამოწმე `git push` ხელით ხელსაწყოს გარეთ.

## თუ გჭირდება ერთი ჯგუფის რეპოების სინქი

გამოიყენე როცა ერთი გუნდი ან პროდუქტი იყო აქტიური და გინდა მხოლოდ
ის არეალი განახლდეს.

**ნაბიჯები**

1. `Main Menu` → `Project Groups List` → (აარჩიე ჯგუფი) → `Sync One Group All Projects Gits V2`

**მოსალოდნელი შედეგი**

* იგივე პერ-რეპო ანგარიში, რაც სრულ სინქში, ოღონდ შეზღუდული არჩეული
ჯგუფის პროექტებით

**ხშირი შეცდომები**

* **ჯგუფთაშორისი დამოკიდებულებები.** თუ ჯგუფის პროექტი დამოკიდებულია
სხვა ჯგუფის გაზიარებულ კოდზე, მხოლოდ ერთი ჯგუფის სინქი დაგვტოვებთ
მოძველებული საერთო კოდით. ეჭვის შემთხვევაში სრული Sync უფრო უსაფრთხოა.

## თუ გჭირდება ერთი პროექტის სინქი (scaffold seeders-ით)

გამოიყენე როცა ერთ პროექტზე მუშაობ და გინდა უფრო მკაცრი ციკლი, ვიდრე
ჯგუფის დონის სინქი.

**ნაბიჯები**

1. `Main Menu` → `Project Groups List` → (ჯგუფი) → `Projects List` →
(პროექტი) → `Sync One Project All Gits With Scaffold Seeders V2`

**მოსალოდნელი შედეგი**

* პროექტის ყველა რეპო სინქდება, პლუს მასთან დაკავშირებული scaffold
seeder რეპოები

**ხშირი შეცდომები**

* **გამოტოვებული scaffold seeder ფოლდერი.** თუ კონფიგურირებული
scaffold seeder სამუშაო ფოლდერი არ არსებობს, ეს ლოგდება როგორც
warning, არა შეცდომა. ძირითადი რეპოები მაინც სინქდება. შეამოწმე ლოგი
თუ scaffold-ის შედეგი გელოდა.

## თუ გჭირდება პროექტზე რეპოს დამატება, ნახვა ან წაშლა

გამოიყენე ახალი პროექტის bootstrap-ისთვის ან ახალი რეპოს onboarding-ისთვის.

**ნაბიჯები**

1. `Main Menu` → `Project Groups List` → (ჯგუფი) → `Projects List` →
(პროექტი) → `Git` ქვემენიუ
2. ქვემენიუდან აარჩიე `New Git` დასამატებლად, ან არსებული ჩანაწერი
წაშლა/რედაქტირებისთვის

**რას სთავაზობს ქვემენიუ**

* Git clone ფაილების load და save (რეპოს სიის export/import)
* ახალი Git ჩანაწერის დამატება (URL, ლოკალური ფოლდერის სახელი, branch
ინფო)
* პერ-რეპო ქმედებები თითოეულ უკვე რეგისტრირებულ ჩანაწერზე

**ხშირი შეცდომები**

* **URL შეუსაბამობა.** თუ ცვლი რეპოს remote URL-ს ამ ქვემენიუში, მაგრამ
ლოკალური ფოლდერი უკვე სხვა remote-ზე იყურება, სინქი აღმოაჩენს
შეუსაბამობას და გაჩერდება. ან წაშალე ლოკალური ფოლდერი ხელით ან
განაახლე remote URL არსებულ checkout-ში.
* **ფოლდერის შეჯახება.** ორი ჩანაწერი, რომელიც ერთსა და იმავე
ლოკალურ ფოლდერზე იყურება, ერთმანეთს დააფარავს clone-ის დროს. ხელსაწყო
ვერ აღმოაჩენს ამას — ფოლდერების სახელები განცალკევებული შეინახე.
